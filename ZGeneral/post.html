Overview
Software Quality Assurance Group (SQAG) reviews technical documents (SRS, SDD, ICD).
Current manual review is slow, inconsistent, and suffers from template errors.
This project proposes an AI-assisted screening system to standardize reviews and reduce burden.
Problem Statement
  • Time-Consuming: Manual verification is repetitive and slow.
  • Inconsistency: Varied document formats and terminology.
  • Visual Blindspots: Images/Diagrams are not machine-readable.
  • High Fatigue: Cognitive load leads to human error.
Objectives
  • Develop an LLM-based web app for SRS verification.
  • Build a modular AI pipeline (Cleaning ‚Üí Mapping ‚Üí Checking).
  • Multimodal understanding of diagrams using Gemma 3.
  • Semantic section mapping to standard SQAG templates.
Motivation
To ensure faster compliance checks aligned with RCI standards while maintaining a "Human-in-the-Loop" approach. The system assists, not replaces, the reviewer.
Proposed Solution
An end-to-end pipeline that cleans PDFs, converts images to text, maps sections semantically, and evaluates checklist compliance using LLM reasoning.

System Implementation

Tech Stack: Python, Streamlit, Docling, Gemma 3, GLM-4.

Workflow: The user uploads a PDF. The system parses it, captions images, maps sections to the SQAG template, runs the checklist, and outputs a scored JSON report.

Challenges
  • Handling irregular SRS templates & noisy layouts.
  • Complex multi-part diagrams.
  • Preventing LLM hallucination (Grounding).
Future Scope
  • Support for SDD, ICD, and STR documents.
  • RAG-based verification for complex queries.
  • Automatic Template Detection.
Conclusion
Successfully developed a modular LLM-based verification system. It significantly reduces manual effort and demonstrates the practical use of Multimodal LLMs in defense documentation.
